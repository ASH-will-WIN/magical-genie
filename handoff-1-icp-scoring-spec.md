# Handoff 1: ICP Theme-Matching & Scoring Modules (Standalone)

## Context

Magical Genie currently fails to find leads when a news article names a small/niche company — Apollo can't resolve that single company, and the whole campaign returns zero leads. The fix is a "Venn diagram" strategy: instead of searching Apollo for the *named* company, extract the underlying market **theme** from the article, use that theme to search for a broad set of **candidate companies**, then score each candidate against Reinvent's ICP and keep only good fits. This produces ~15-20 qualified companies per article instead of 1 company that may not resolve.

## Scope of this handoff

Build and test the following as **standalone, independently runnable modules** — not wired into `main.py`, `apollo.py`, or the database yet. No pipeline changes, no schema migrations, no Streamlit UI changes in this handoff. The goal is to validate the theme-extraction and scoring logic in isolation, using real inputs, and produce real precision/recall numbers we can use to set thresholds — not to ship a working feature yet.

Do not modify any existing files in `services/`, `main.py`, `frontend.py`, or `database.py`. All new code goes in a new `services/icp_matching/` directory.

## Deliverable 2: Theme extraction module

**File:** `services/icp_matching/theme_extraction.py`

Function: `async def extract_theme(article_text: str) -> ThemeExtraction`

Use OpenAI structured outputs (Pydantic), same pattern as the existing `services/context.py`. Temperature 0.1.

```python
class ThemeExtraction(BaseModel):
    theme_summary: str  # 1-2 sentences on the underlying market shift/problem
    affected_problem_space: list[str]  # e.g. "patient transport logistics"
    vendor_categories_who_benefit: list[str]  # e.g. "AI-powered scheduling platforms"
    apollo_keyword_candidates: list[str]  # 3-6 short phrases suitable for company search
```

**Prompt requirements:**
- Explicitly instruct the model NOT to extract the company named in the article — extract the *category* of vendor who would sell a solution into this situation.
- Include 2-3 few-shot examples in the prompt showing article → theme mapping (write these using realistic HealthcareDive-style headlines, not the real MedHaul/Mediphant/Audact companies as subjects — those are ICP examples, not article examples).
- Validate output schema strictly, retry up to 3 times on failure (match existing pattern in `context.py`).

## Deliverable 3: Pre-filter module

**File:** `services/icp_matching/prefilter.py`

Function: `def prefilter_candidates(candidates: list[dict], icp: dict) -> list[dict]`

Pure deterministic logic, no LLM call. Given a list of Apollo-style company dicts (each with at minimum `name`, `domain`, `industry`, `employee_count`), remove any company whose `industry` (case-insensitive, substring match) appears in `icp["hard_exclude_industries"]`, and any company outside `icp["size_band"]["employee_count_min"/"max"]`. Return the filtered list. Log how many were removed and why (simple print/log statement is fine for this handoff — no need for structured logging yet).

## Deliverable 4: Scoring module

**File:** `services/icp_matching/scoring.py`

Function: `async def score_candidate(company: dict, icp: dict) -> CandidateScore`

```python
class CandidateScore(BaseModel):
    score: int  # 0-100
    reason: str  # one sentence
    exclude: bool
```

Temperature 0.1. Prompt should:
- Take company name, Apollo description, Apollo industry tag as input
- State the ICP definition inline (small-to-mid health-tech SOFTWARE vendors building/adding AI features to their own product — not care-delivery orgs, not payers, not pharma)
- Hard instruction: if this is clearly a hospital, clinic, physician practice, insurance carrier, or pharma company, return score 0 and exclude=true regardless of anything else
- Otherwise score based on fit against `vertical_keywords`, `technology_signal_keywords`, and `buying_trigger_keywords` presence/relevance in the description

Also build: `async def score_candidates_batch(companies: list[dict], icp: dict) -> list[CandidateScore]` using `asyncio.gather` for parallel scoring (match the existing parallelism pattern from `services/copy.py`).

## Deliverable 5: Test harness (this is the actual point of the handoff)

**File:** `services/icp_matching/test_calibration.py`

This is a standalone script (`python test_calibration.py`), not a pytest suite — it's meant to be read by a human.

**Step A — Build a labeled test set.** Hardcode a list of ~12-15 companies in the script:
- Positives (should score high): use real data pulled from these URLs — pass each through a simple scrape (reuse `services/scraper.py`) to get a real description, don't hand-write these:
  - https://gomedhaul.com/
  - https://mediphant.ai/
  - https://www.audacthealth.com/
  - Add 2-3 more real health-tech AI startups you find via a quick web search (Claude Code should search for "small health tech AI startup" type results, grab name + description) to round out the positive set
- Negatives (should score low/zero): a hospital system (e.g. search and grab a real one), a health insurance payer, a pharma company, and — important — one large enterprise health-tech company (500+ employees, e.g. a company like Epic or Cerner-scale) to test that `size_band` is actually doing work, not just industry exclusion.

**Step B — Run the pipeline on the labeled set.** For each labeled company: run `prefilter_candidates`, then `score_candidate` for anything that survives. Print a table: company name | expected (positive/negative) | prefilter result (kept/dropped, and why) | LLM score | LLM reason | exclude flag.

**Step C — Report separation.** Print the min/max/mean score for the positive group vs. the negative group. This is the number we actually care about — we want to see a clean gap (e.g. positives clustering 75-95, negatives clustering 0-40). If there's overlap, that's a signal the prompt needs work, not that we should just pick a threshold that splits the difference.

**Step D — Article-driven test.** Pull 3 real recent HealthcareDive-style articles (Claude Code should web-search for these — "HealthcareDive AI health tech" or similar, grab 3 real article URLs and scrape them with the existing scraper). Run `extract_theme` on each. Print the theme extraction output for all 3 so we can manually judge whether the `apollo_keyword_candidates` look like reasonable search terms — this doesn't need Apollo wired up yet, just print what the theme extraction produces so we can eyeball it.
